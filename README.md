# Jothand
### A Hybrid Shorthand for Fast Notes

## Overview

Jothand ("Jot") is a hybrid shorthand system designed to accelerate personal notetaking without replacing ordinary handwriting.

Its defining feature is selective use. Instead of requiring the writer to use shorthand for everything, Jothand works as a set of **optional** shortcuts that can be blended into normal writing at the user’s discretion. A writer may shorten only the words, sounds, or phrases that are useful in the moment, while leaving the rest in ordinary English.

This makes the system especially practical for real-world notes. It works particularly well with cursive handwriting, since adjoining strokes naturally blend together and make shorthand forms faster to write.

Jot is mostly phonetic: it often drops vowels and focuses on simplified consonants and common consonant blends. Unlike formal shorthand systems, it does not use added dots, ticks, positional marks, or other modifiers to indicate vowels. This means that Jothand is best interpreted by the person who originally wrote it, and has some memory/context for filling in missing sounds and details (hence why it's more approprate for personal notetaking than other forms of communication).

The system also includes a set of single-character word abbreviations for common words, and some of these can be chained together to represent short phrases such as `ue` for “of the.”

## What Makes Jothand Different

Unlike formal shorthand systems, Jot is not meant to replace all written English. It is a system of optional shortcuts that can be inserted into normal writing wherever the user chooses.

It is especially useful for:

- class notes
- meeting notes
- outlines
- personal reminders
- fast handwritten drafting

Because it is hybrid, proper names, numbers, and any material where clarity matters can simply be written as-is. This also partly compensates for the fact that the system does not provide for vowel sounds, capitalization, or punctuation - user is free to include those where context (or comfort) is at stake.

## Core Principles

1. **Hybrid use** — The writer can switch freely between normal English and shorthand.
2. **Cursive-friendly** — The system works especially well in cursive, where strokes naturally blend together.
3. **Mostly phonetic** — Words are compressed by emphasizing consonants and consonant blends.
4. **No vowel markers** — Vowels are usually omitted and recovered from context.
5. **No modifiers** — The system does not depend on extra dots, ticks, shading, or positional rules.
6. **Whole-word abbreviations** — Common words can be written with a single letter or symbol.
7. **Phrase chaining** — Abbreviations can be combined to represent common short phrases.
8. **Plain-text friendly** — The system can be documented digitally using standard Unicode characters and symbols.
9. **Preserve boundary vowels** — Words that begin or end with a prominent vowel sound are often clearer when the vowel is written out. For example, write `only` rather than `_l`, and `i⏐.lf` rather than `⏐.lf` for "itself."

## Categories of Jothand Forms

Jothand uses several kinds of shortening:

### 1. Ordinary writing kept as-is
Some text is simply written in normal English. This is common for:

- proper names
- numerals
- unusual words
- anything where immediate readability matters

### 2. Sound substitutions
Certain consonants or blends are replaced with faster symbols.

### 3. Whole-word abbreviations
Very common words can be reduced to a single character or symbol.

### 4. Phonetic symbol substitutions
Some words are replaced by numbers or symbols because they sound alike.

## Core Sound Symbols

Most consonants are written normally. The following sounds use special shortened forms.

| Sound / Letter | Jot Form | Notes |
|---|---:|---|
| t | `⏐` | Uncrossed handwritten `t`; represented digitally as vertical pipe |
| th | `t` | Full crossed `t` |
| m | `⌒` | Arched stroke |
| w | `◡` | Inverted arched stroke, much wider than a normal `u` shape |
| s / hard c | `.` | Unvoiced “s” sound, as in *see* or *cent* |
| soft s | `˙` | Dot-above character for voiced "z" sound - also, used to pluralize words |
| k / hard c | `c` | “k” sound, as in *cat* |
| n | `_` | Wide underscore stroke; should be distinct from a connector line |
| ch | `(` | |
| sh | `)` | |
| ing | `,` | Used where the pronunciation is “ing” |
| nd / nt / and | `+` | Can function as blend or as a whole-word abbreviation |
| at | `ɑ` | Used for the sound or the word “at” |

> **Note:** When the shorthand form of a sound is no shorter than the original letter, prefer the original. For example, `q` already represents the k+w sound in English and should be kept as `q` rather than written as `c◡`.

## Chained Abbreviations

One of the useful features of Jothand is that common word abbreviations can be chained together to form short "phrases". Chaining is only valid when **every word in the sequence is represented by a single abbreviation** — multi-character phonetic abbreviations (like `lc` for *like* or `hv` for *have*) cannot be run together with adjacent abbreviations and must be separated by a space.

Examples:

- `ue` = **of the**
- `s∅1ue` = **is not one of the**
- `2br∅2b` = **to be or not to be**
- `ɑse` = **that is the**

Chaining should be used within reason. The goal is speed without making later reading difficult.

## Reading Strategy

Jothand assumes that many omitted elements can be recovered from context. In practice, this usually includes:

- omitted vowels
- whether a symbol is functioning as a sound or a whole word
- distinctions such as `is` vs `as`, or `at` vs `that`
- whether a symbol is part of a chain or a separate word

Because the system is intended for practical note-taking, readability depends on familiarity, surrounding words, and the writer’s judgment.## Ambiguity

Jot tolerates more ambiguity than formal systems like Gregg shorthand. This is by design: the system is personal, and the writer always has the option to write more clearly when context demands it.

In a chain like `2br∅2bɑse`, the non-alphabetical characters make it reasonably clear that a sequence of word abbreviations is being used. But in plainer sequences, ambiguity can arise. For example, `be` could be read as:

- plain English: *be*
- a phonetic abbreviation: *bear*, *bill*, *by*, etc. (consonants only, vowel inferred)
- two chained word abbreviations: *b* (be) + *e* (the) → **be the**

The writer must decide how much to compress based on whether the result will be recoverable from context. There is no rule that forces abbreviation; using full words or partial abbreviations where needed is always the right choice.

## What Jothand Does Not Use

Jothand deliberately leaves out several features found in more formal shorthand systems:

- capitalization
- explicit vowel markers
- symbol modifiers
- positional rules

Several common punctuation marks — including `,` (comma), `.` (period), `(` and `)` (parentheses), and `-` (hyphen) — overlap with Jothand symbols. Using them as grammatical punctuation creates ambiguity and is discouraged. In practice, prose punctuation should be limited to `!` and `?`, which do not conflict with any Jothand form.

These constraints make the system simpler and faster, especially for informal notes.

## Why Cursive Works Best

Although Jothand can be used alongside print writing, cursive generally works better because strokes flow together more naturally. This makes it easier to insert shorthand forms directly into ordinary words and phrases without interrupting writing rhythm.

In that sense, Jothand is not just a code or symbol set. It is a practical layer of compression that works best when integrated into the writer’s natural hand.

## Background

I first encountered *Wesley’s Short-cut Shorthand* around 1990, in an elementary school library. I quickly adopted a number of its underlying ideas - what I could learn and remember in the two weeks before my book was due back - especially its simplified approach compared to more formal systems such as Gregg shorthand.

Over time, I modified the system for my own use, dropped some features, and introduced new ones. The version documented here reflects a personally developed shorthand that has been used in practice for more than three decades.
